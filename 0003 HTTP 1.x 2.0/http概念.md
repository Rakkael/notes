1、什么是http协议？
------
```
互联网服务器 传输超文本 到本地浏览器 的超文本传输协议。
是基于TCP的传输协议。

http是无连接的。也就是每次连接只处理一个事件，当事件处理完毕收到客户端的应答后就断开连接了。
http是媒体独立的。如果服务端和客户端知道如何处理传输的数据，那么任何数据都可以通过http传输。
http是无状态的。如果后面的事件处理需要前面传输的数据，那么就需要重传。这样缺点是传输数据量增大。优点是如果后面事件不需要前面的数据，那可以节约应答时间。

无状态举例：登录商城加购买东西。 登录输入一次账号密码。成功后服务器忘记你是谁，是否登录过。加购时又要输入账号密码。
解决方法：seesion cookie 
cookie 保存密码： 1、 账号密码一起放入cookie 2、密码加密与账户放入cookie 3、账号存入account 账号与秘钥MD5加密存入ssid 下次直接验证加密规则

cookie 和 session 对比
cookie 在 http 协议中定义在 header 中的字段。
session 是抽象概念、~~开发者为了实现中断和继续等操作~~，将client和server之间一对一的交互，抽象为“会话”，进而衍生出“会话状态”，也就是 session 的概念。
即session描述的是一种通讯会话机制，而cookie只是目前实现这种机制的主流方案里面的一个参与者，它一般是用于保存session ID。
```

3、说一下请求行、请求头、空行、请求数据
------
```
请求行是由三个部分组成的，首先是请求方法，get，post之类的。然后是URL链接字段，最后是http版本

get请求获取指定的资源 浏览网站点击链接都是get方法
post 提交数据进行处理请求，上传文件，提交表单都是post
head和get有点像，但是head请求只传输响应头不传输页面数据，主要用于获取报头


常用请求头 key:value的形式
第一个是user Agent  产生请求的浏览器类型
第二个是accept 客户端 识别的 内容列表。
第三个是host 主机名

空行
空一行是表明请求头结束，下面没有请求头了

请求数据
不适用于get，而适用于post，例如填写表单的时候的post就有请求数据

```
>重点 get 与 post 区别

GET 与 POST 的区别
------
```
GET语义 请求获取 指定的 资源，幂等、可以缓存的。  
POST语义 请求对提交的资源做出处理，不幂等，不可缓存。
GET 数据附在URL后 POST数据存在HTTP包体中
GET 产生一个数据包 （header data一起发）
POST产生2个 先发header 响应100 再发 data

什么是幂等？多次操作 返回同样结果 不修改状态信息 可重复执行。 非幂等：最终结果与次数有关，每次结果不同。

```
4、http响应消息结构
------
```
响应消息和http请求结构很像，也是三部分。
第一个是状态行，然后是响应头，空行，响应正文。

状态行包含 http版本 + 状态码 + 状态码的描述
响应头 content-type 内容类型 content-length 内容长度
```

5、状态码都有哪些？
-------
````
200 ok 没问题通过了
400 bad request 我的语法有问题
404 not found 找不到资源
403 啥都好着但是被拒了
405 请求方法被禁了
408 等待时间过长 超时了
502 代理服务器收到了一个无效请求
504 代理服务器 超时获取远端请求
505 不支持 http版本

1开头 继续操作
2开头 正常
3开头 重定向 进一步操作
4开头 客户端错误
5开头 服务器错误
````
